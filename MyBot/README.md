# MyBot 智能团队任务协同助手 - 使用说明

## 📖 这是什么？

MyBot 是一个**团队任务管理 Web 应用**，帮助团队负责人：
- 创建业务需求
- 用 AI 自动将需求拆解为任务
- 智能推荐任务分配给合适的成员
- 通过看板追踪任务进度
- 自动生成管理报告

简单来说：**需求 → AI 拆解 → 分配 → 看板追踪 → 报告**

## 🚀 如何启动项目

### 第一步：安装必备软件

你的电脑需要先安装以下软件：

| 软件 | 用途 | 下载地址 |
|------|------|----------|
| **Node.js** (v18+) | 运行 JavaScript | https://nodejs.org/ （选 LTS 版本） |
| **PostgreSQL** (v14+) | 数据库 | https://www.postgresql.org/download/ |
| **Git** | 代码管理 | https://git-scm.com/download/win |

> 💡 安装 Node.js 时勾选"自动安装必要工具"选项。
> 💡 安装 PostgreSQL 时记住你设置的密码（默认用户名是 `postgres`）。

### 第二步：创建数据库

1. 打开 **pgAdmin**（安装 PostgreSQL 时自带的）或命令行
2. 创建一个名为 `mybot` 的数据库：

```sql
CREATE DATABASE mybot;
```

或者用命令行：
```bash
psql -U postgres -c "CREATE DATABASE mybot;"
```

### 第三步：配置后端

1. 打开终端，进入后端目录：
```bash
cd d:\CaoLing\YourBot\backend
```

2. 安装依赖（第一次运行需要，以后不用）：
```bash
npm install
```

3. 修改 `.env` 文件中的数据库密码：
```
DATABASE_URL="postgresql://postgres:你的密码@localhost:3001/mybot?schema=public"
```
> ⚠️ 把 `你的密码` 替换成你安装 PostgreSQL 时设置的密码

4. 初始化数据库表结构：
```bash
npx prisma migrate dev
```
> 这会自动创建所有需要的数据库表

5. 生成 Prisma 客户端：
```bash
npx prisma generate
```

6. 修改 `.env` 中的端口为 3001（因为前端占用了 3000 端口）：
```
APP_PORT=3001
```

7. 启动后端服务：
```bash
npm run start:dev
```

看到类似这样的输出就说明成功了：
```
Application is running on: http://localhost:3001/api/v1
```

### 第四步：配置前端

1. **新开一个终端窗口**（不要关闭后端的），进入前端目录：
```bash
cd d:\CaoLing\YourBot\frontend
```

2. 安装依赖：
```bash
npm install
```

3. 启动前端开发服务器：
```bash
npm run dev
```

看到类似这样的输出就说明成功了：
```
  VITE v8.x.x  ready in xxx ms

  ➜  Local:   http://localhost:3000/
```

### 第五步：打开浏览器

在浏览器中访问：**http://localhost:3000**

你应该能看到登录页面了！🎉

## 👤 用户角色说明

系统有三种角色：

| 角色 | 能做什么 |
|------|----------|
| **管理员 (admin)** | 管理用户、拥有所有权限 |
| **经理 (manager)** | 创建需求、分配任务、查看看板和报告 |
| **成员 (member)** | 查看和更新自己的任务状态 |

### 创建第一个管理员账号

系统刚启动时没有任何用户，你需要手动在数据库中创建一个管理员。

打开终端，运行以下命令（需要先安装 bcrypt 工具或使用 Node.js）：

```bash
cd d:\CaoLing\YourBot\backend
node -e "const bcrypt = require('bcrypt'); bcrypt.hash('admin123', 10).then(h => console.log(h))"
```

复制输出的密码哈希值，然后在 PostgreSQL 中执行：

```sql
INSERT INTO users (username, password_hash, name, role, email, status, created_at, updated_at)
VALUES ('admin', '上面复制的哈希值', '管理员', 'admin', 'admin@mybot.com', 'active', NOW(), NOW());
```

然后你就可以用以下账号登录了：
- 用户名：`admin`
- 密码：`admin123`

> 登录后可以在"用户管理"页面创建更多用户，不需要再手动操作数据库。

## 📱 功能使用指南

### 1. 登录系统

打开 http://localhost:3000，输入用户名和密码，点击"登录"。

登录后会根据角色自动跳转：
- 管理员 → 用户管理页
- 经理 → 仪表盘
- 成员 → 我的任务

### 2. 创建需求（经理/管理员）

1. 点击左侧菜单"需求管理"
2. 点击右上角"新建需求"
3. 填写表单：
   - **标题**：需求的简短名称
   - **背景**：为什么要做这个需求
   - **目标**：做完后要达到什么效果
   - **交付物**：具体要交付哪些东西（每行一个）
   - **优先级**：紧急/高/中/低
   - **截止日期**：什么时候要完成
4. 点击"创建"

### 3. AI 拆解需求（经理/管理员）

1. 在需求列表中，找到状态为"草稿"的需求
2. 点击进入需求详情，点击"AI 拆解"
3. 系统会自动调用 AI 将需求拆解为任务树
4. 你可以：
   - 点击任务节点编辑内容
   - 修改预估工时、描述等
5. 确认无误后点击"确认任务树"保存

### 4. 分配任务（经理/管理员）

1. 进入"任务分配"页面
2. 勾选需要分配的任务
3. 点击"AI 智能推荐"获取分配建议
4. AI 会根据成员技能、工作量等推荐最合适的人选
5. 你也可以手动选择成员
6. 确认后点击"确认分配"

### 5. 更新任务状态（成员）

1. 进入"我的任务"页面
2. 看到分配给你的所有任务
3. 点击状态按钮更新：
   - **开始** → 任务从"待办"变为"进行中"
   - **完成** → 任务标记为完成
   - **阻塞** → 需要填写阻塞原因
   - **延期** → 标记任务延期

### 6. 查看看板（经理/管理员）

1. 进入"任务看板"页面
2. 看到所有任务按状态分列显示：待办 | 进行中 | 阻塞 | 已完成 | 延期
3. 可以按需求或负责人筛选
4. 看板会实时更新（有人更新任务状态时自动刷新）

### 7. 生成报告（经理/管理员）

1. 进入"报告管理"页面
2. 点击"生成报告"
3. 选择报告类型（日报/周报/阶段报告）和日期范围
4. AI 会自动生成报告内容
5. 你可以编辑生成的报告
6. 支持导出为 Markdown 格式

### 8. 查看仪表盘（经理/管理员）

仪表盘显示：
- 需求总数、进行中任务、已完成任务、逾期任务
- 任务状态分布图
- 成员工作负载
- 活跃告警（逾期、阻塞、超载）

### 9. 管理用户（管理员）

1. 进入"用户管理"页面
2. 可以：
   - 创建新用户（设置用户名、密码、角色）
   - 修改用户角色
   - 禁用用户（禁止登录但保留数据）
   - 管理成员档案（技能、工作量等）

## ⚠️ 常见问题

### Q: 启动后端报错 "Cannot connect to database"
**A:** 检查以下几点：
1. PostgreSQL 服务是否已启动
2. `.env` 文件中的数据库密码是否正确
3. `mybot` 数据库是否已创建

### Q: 前端页面打不开
**A:** 确认：
1. 前端服务是否在运行（终端有没有报错）
2. 浏览器访问的是 http://localhost:3000（不是 3001）

### Q: AI 功能不工作
**A:** 当前 AI 功能使用的是模拟数据（MVP 阶段）。如果要接入真实 AI：
1. 安装 Ollama：https://ollama.ai
2. 下载模型：`ollama pull qwen2.5`
3. 确保 `.env` 中的 `AI_MODEL_ENDPOINT` 指向正确地址

### Q: 登录后看到"未授权"页面
**A:** 你的账号角色没有权限访问该页面。不同角色能访问的页面不同，参考上面的角色说明。

### Q: 如何停止服务？
**A:** 在运行服务的终端窗口按 `Ctrl + C`。

## 💡 小贴士

1. **第一次使用**建议先创建几个测试用户（一个经理、几个成员），体验完整流程
2. **Prisma Studio** (`npx prisma studio`) 是一个很好用的数据库管理工具，可以直接在浏览器中查看和编辑数据
3. 后端端口是 **3002**（需要在 .env 中设置 `APP_PORT=3002`），前端开发服务器端口是 **3000**，前端会自动把 API 请求代理到后端
4. 所有 API 都在 `/api/v1/` 路径下，可以用 Postman 或浏览器直接测试
